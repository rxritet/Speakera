Подготовить проект к полноценному мобильному запуску, зафиксировать переходную схему Firebase, запланировать новые функции и обновить визуальный язык приложения: две темы, более современная палитра и больше анимаций.

## Статус проверок на сейчас (без full mobile run)

### Что проверено и подтверждено
- Flutter toolchain, зависимости, анализ и тесты: OK.
- `flutter analyze`: без ошибок.
- `flutter test`: проходит.
- `server/dart analyze`: без ошибок.
- Web-клиент запускается на 8081.
- `API_BASE_URL` поддерживается через `--dart-define`.
- Для Web реализован fallback общего backend host (`scheme://current-host:8080`).
- Добавлен единый скрипт запуска dev-окружения одной командой: `scripts/dev-web.ps1`.

### Что блокирует end-to-end сейчас
- Backend на `localhost:8080` не поднимается, потому что Docker daemon недоступен
	(`dockerDesktopLinuxEngine` недоступен).
- Локальный PostgreSQL как сервис отсутствует, поэтому запуск server вне Docker
	сейчас невозможен без отдельной установки БД.

### Важные риски перед полноценным мобильным запуском
- Android release сейчас подписывается debug signing config (нужно заменить).
- Firestore rules сейчас открыты (`allow read, write: if true;`) и требуют
	ужесточения до production-уровня.
- Firestore indexes пока пустые, нужно добавить составные индексы под реальные
	запросы после стабилизации схемы.

## Приоритет 2: Firebase-связь и миграция данных
1. Зафиксировать Firebase как целевую платформу для auth, профиля, дуэлей, check-ins и пушей.
2. Согласовать модель Firestore.
3. Подготовить экспорт данных из PostgreSQL.
4. Подготовить импорт в Firestore батчами.
5. Добавить правила доступа и индексы Firestore.
6. Подготовить постепенное переключение чтения и записи.

### Текущий статус приоритета 2
- Документ миграции есть: `FIREBASE_MIGRATION.md`.
- Экспорт PostgreSQL реализован: `server/bin/export_postgres_for_firebase.dart`.
- Импорт в Firestore батчами как отдельный executable пока отсутствует.
- Правила и индексы Firestore подключены, но пока в dev-состоянии.

### Целевая схема Firebase
- `users/{userId}`: профиль, счетчики, legacy id.
- `duels/{duelId}`: дуэли, статусы, таймстемпы.
- `duels/{duelId}/participants/{userId}`: участники и стрики.
- `duels/{duelId}/checkins/{checkinId}`: история отметок.
- `users/{userId}/badges/{badgeId}`: бейджи.
- `users/{userId}/devices/{deviceId}`: FCM токены и platform metadata.

### Порядок миграции
1. Freeze writes на старом backend.
2. Экспорт users.
3. Экспорт duels.
4. Экспорт participants.
5. Экспорт checkins.
6. Экспорт badges.
7. Импорт в Firestore.
8. Сверка counts и контрольных записей.
9. Dual-read период.
10. Переключение записи на Firebase.
11. Деактивация PostgreSQL как основной базы после стабилизации.

## Приоритет 3: новые функции
### Функции с высоким приоритетом
- Push-уведомления через FCM.
- Приватные дуэли по инвайту.
- Система друзей.
- Оффлайн-кэш и очередь синхронизации.
- История прогресса по дням и неделям.
- Расширенные бейджи и достижения.

### Функции со средним приоритетом
- Еженедельные челленджи.
- Серии с визуальным прогрессом на календаре.
- Мини-лента активности друзей.
- Улучшенный экран профиля с аналитикой.
- Deep links на приглашения и дуэли.

### Функции с более низким приоритетом
- Групповые дуэли 3+ игроков.
- Тематические сезоны.
- Реферальная система.
- Глобальные турниры и события.

### Новые функции (предложение)

1. Групповые дуэли 3-10 игроков с weekly scoring
- Тип дуэли: `group`, таблица участников с очками за check-in и штрафами за пропуски.
- Режимы: private invite link и public lobby.
- Завершение: авто-ранжирование, награды топ-3, история результатов.

2. Интеграции здоровья и активности
- Apple Health / Google Fit / Health Connect для автоподтверждения привычек
	типа шагов, сна, тренировок.
- Режим "trusted check-in": событие валидируется внешним источником,
	меньше ручного фрода.

3. Мониторинг дуэлей v2 (anti-cheat + SLA оповещений)
- Timeline-события дуэли: create, accept, check-in, miss, streak-break.
- Детектор аномалий: слишком частые check-in, backfill-попытки, рассинхрон времени.
- Метрики стабильности: delivery rate пушей, median latency обновлений,
	percent of missed check-ins.

4. Коуч-ассистент привычек
- AI-резюме прогресса за неделю, персональные рекомендации
	(нагрузка, частота, время check-in).
- Умные напоминания с учетом локального графика пользователя.

## Приоритет 4: новый дизайн
### Визуальное направление
- Уйти от стандартного Material default look.
- Использовать более выразительную палитру и контраст.
- Сохранить удобство и читаемость на мобильных экранах.
- Сделать интерфейс более живым за счет глубины, градиентов и движения.

### Две темы
#### Светлая тема
- Светлая база с тёплым акцентом.
- Основной акцент: оранжево-янтарный и графитовый.
- Фон: мягкие градиенты, не чисто белый.
- Карточки: легкая тень, аккуратные бордеры.

#### Тёмная тема
- Глубокий графитовый/синий фон.
- Акцент: огненно-оранжевый, бирюзовый для вторичных статусов.
- Карточки: более высокий контраст, подсветка активных элементов.
- Не делать “просто инверсию” светлой темы.

### Анимации
- Плавное появление экранов.
- Staggered reveal для карточек дуэлей и списков.
- Анимация прогресса стрика.
- Мягкий переход при смене тем.
- Акцентные micro-interactions для кнопок check-in, create duel и accept duel.
- Анимация empty states, чтобы они не выглядели статично.

### Что обновить в UI
- Splash / login / register.
- Главный экран дуэлей.
- Экран дуэли.
- Профиль.
- Лидерборд.
- Экран настроек.
